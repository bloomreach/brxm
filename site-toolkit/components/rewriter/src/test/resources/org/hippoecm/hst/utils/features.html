Hippo Repository / DMS

Why use Hippo ECM?

Managing all information in an organization is a daunting task. Knowledge is stored in numerous files, emails, webpages, forms, Excel sheets, and so on, and is handled differently for every organization. To work with all this information, every organization employs a diverse set of applications that adds logic and behaviour. Hippo ECM can integrate those applications, manage the underlaying information and present the end user with a simple, unified environment for all their information management.

Why choose Hippo ECM?

  • Hippo ECM, 100% Open Source Enterprise Content Management: Hippo ECM facilitates an open and flexible way of using your information by following internationally accepted open standards. This allows integration of other applications, ensures freedom to change to other ECM systems, if so desired
  • Extensively Modular: Hippo EMC differentiates itself from other ECM and CMS systems by its clean separation of content storage, management and presentation. These modules (independent units) are even subdivided into smaller modules. This extensive modularity allows for fast development and for extensibility, even by other parties. An overview of the higher level modules and their interconnection can be found at ECM Architecture .

Faceted Navigation

Hippo Facet Navigation works out of the box. Browsing through authorized faceted navigation gives you multiple ordered views with the number of occurences on your (unstructured) physical data. One single document can be found via multiple faceted paths. Users do not have to think about structuring the content in a hierarchical of taxonomic order, but rather the structure evolves organically based on meta data and the users' authorisation.

The Hippo Facet Navigation works extremely fast for a faceted view with counting and authorization. The view renders as if you are clicking through a cached File System.
Hippo Facet Navigation includes user based faceted authorisation which makes it very powerful as a DMS, where instead of users having to worry about structure, they browse to their specific content the way they want.
